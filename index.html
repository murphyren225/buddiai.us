01 / CLOUD

云端处理

设备端只做拾音与播放，ASR、LLM、TTS 全部在云端完成。硬件门槛与成本最低，适合快速量产。

ARCH-01

轻终端 + 全云管线

ESP32-S3 微雪 ESP32-S3 Touch AMOLED

本地接收设备采用 ESP32-S3 或微雪（Waveshare）系列开发板：双核 240MHz、内置 Wi-Fi，跑轻量唤醒词检测（如 ESP-SR），随后将音频流式上传云端。整条智能管线跑在服务器上，模型能力上限最高，OTA 升级也只需要改云端。

MIC · ESP32-S3→ Wi-Fi / WebSocket→ 云端 ASR→ 云端 LLM→ 云端 TTS→ SPK · 设备播放
  • 单机 BOM 最低（几十元级），无本地算力要求
  • 模型可随时换最强云端大模型，能力天花板高
  • 延迟受网络 RTT 与云端排队影响，存在波动（±300ms 常见）
  • 断网即失能；语音数据需上云，隐私合规成本高

典型延迟画像 *

网络上行 + RTT80–200 ms

端到端首响1.0–1.8 s
离线可用性不可用
02 / HYBRID

端云协同 · Hub 模式

在家庭/场所内放一台 Hub，敏感环节（ASR、记忆、隐私过滤）本地完成，重计算按需上云。一台 Hub 服务多台轻终端。

ARCH-02

本地 Hub + 云端增强

终端: ESP32-S3 Hub: RK3588 / Apple Silicon 级

终端仍是低成本 ESP32 设备，但音频先到局域网内的 Hub：本地完成 ASR 与会话记忆，语音永不出户；LLM 可跑本地小模型，也可只把脱敏后的文本发云端换取大模型能力。局域网一跳（~1ms）替代公网往返，延迟稳定可控。

MIC · ESP32-S3→ 局域网 ~1ms→ Hub: 本地 ASR + 记忆→ 本地 LLM / 脱敏上云→ TTS→ SPK
  • 实测：本地 ASR ~300ms，全管线稳态 ~1.45s（Intel CPU 级 Hub，热路径）
  • 原始语音不出家门，只有脱敏文本可选上云——隐私结构性优势
  • 断网时核心对话仍可用（本地小模型兜底）
  • 一台 Hub 带多个终端，边际成本随设备数摊薄

典型延迟画像 *

局域网传输~1 ms
本地流式 ASR（实测）~300 ms
LLM TTFT（本地 3B 级）250–500 ms
端到端首响（实测热路径）~1.45 s
离线可用性核心可用
03 / EDGE

纯端侧推理

整条管线塞进设备本体，零网络依赖。用 Jetson 或香橙派的 GPU/NPU 做本地推理加速，适合对离线、实时性、数据主权要求苛刻的场景。

ARCH-03

单机全本地管线

Jetson Orin Nano Orange Pi 5 (RK3588)

Jetson Orin Nano（最高 67 TOPS，CUDA 生态）或香橙派 Orange Pi 5（RK3588，6 TOPS NPU，性价比路线）直接承载 ASR + LLM + TTS 全栈。没有任何网络环节，延迟只由本机算力决定——也因此最稳定、最可预测。

MIC→ 本地 ASR→ 本地 LLM (GPU/NPU 加速)→ 本地 TTS→ SPK
  • 零网络依赖：无 RTT、无抖动、无云服务账单，延迟方差最小
  • 数据 100% 不离机，天然满足最严数据合规
  • Jetson 路线：3B–7B 模型流畅（~20–40 tok/s）；香橙派路线：1.5B–3B 量化模型
  • 整机成本与功耗最高（Jetson 千元级 / OPi5 数百元级），散热需设计

端到端首响0.9–1.6 s
离线可用性完全离线

05 / SOLUTION

怎么选？

☁️ 选云端处理，如果你要快速出货、极致压低单机成本、需要旗舰大模型能力，且用户场景默认有稳定网络。
🏠 选端云协同，如果你要隐私作为产品卖点（语音不出户）、多设备共享一台算力、断网也要能用——延迟与成本的最佳平衡点。
⚡ 选纯端侧，如果你要完全离线运行、毫秒必较的确定性延迟、数据绝对不离机（工业、车载、安防、涉密场景）。
